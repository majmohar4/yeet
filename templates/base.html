📖 yeet — how to use

The basics

  • Drop files anywhere on the upload area, or click to browse, or paste with Ctrl+V.
  • On a phone or tablet use the 📋 Paste, 🖼 Photo, and 📁 Folder chips inside the drop zone.
  • Once uploaded you get a /f/<id> link copied to your clipboard automatically.
  • Files expire automatically. Pick 1h, 24h, or 7d for each upload.

🔥 Burn after read

Tick Burn after read to make a single-view link. The file is wiped from disk the instant someone downloads it — no archive, no second chance. Atomic claim prevents two people from racing the same link.

📁 Folder uploads

Drag a whole folder onto the drop zone, or click 📁 Folder. You get one share link /b/<id> with a file list and a Download all (.zip) button. Each file is still scanned individually.

📋 Public clipboard

The Public clipboard panel on the home page is a global feed — every visitor sees every recent paste. Items you posted are tinted green and tagged YOU; only you can pin or delete them. Don't put secrets in there.

🔒 Password protection

Type a password into the lock field before uploading. Recipients enter it to unlock the download. Passwords are hashed with bcrypt — yeet never stores the raw value.

🤖 CLI

yeet upload photo.jpg
yeet upload secret.pdf --password hunter2
yeet upload one-shot.zip --burn

# Plain-text URL — handy for piping
curl -F f=@build.log https://yeet.majmohar.eu/raw

What you can upload

The check is permissive: ~1000 named extensions across every common category, plus a fallback that accepts any reasonable extension shape. Files without an extension also work.

⚠ Executables (allowed but flagged)

Windows .exe .msi .dll .bat .ps1 .vbs, Unix .sh .so .deb .rpm, macOS .dmg .app, Java .jar — all upload fine and get a ⚠ badge. Inline previews are refused; only the /f/ download path serves them and always as an attachment.

📄 HTML / SVG / JS

Accepted, but yeet shows them as plain text — no rendering, no script execution, no fake-login phishing. If you want the rendered version, download and open locally.
